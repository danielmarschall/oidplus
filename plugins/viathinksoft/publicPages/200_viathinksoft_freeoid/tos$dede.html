Nutzungsbedingungen

  • Dieser Dienst soll hauptsächlich Privatpersonen, kleinen Arbeitsgruppen und Softwareentwicklern von Freeware, Shareware oder Open-Source-Software die Möglichkeit geben, eine kostenlose Objektkennung (OID) mit minimalem bürokratischem Aufwand zu erhalten.
  • Sie müssen lediglich eine gültige E-Mail-Adresse eingeben, an die ein Aktivierungslink gesendet wird. Um die OID zu erhalten, müssen Sie auf den Link in der automatisch generierten E-Mail klicken und dann Ihren Namen eingeben. Dies ist das einzige Pflichtfeld. Während des gesamten Prozesses wird kein menschlicher Eingriff von unserer Seite durchgeführt, und die OID wird sofort zugewiesen.
  • Nur OIDs, die auf dieser Registrierungswebsite registriert und angezeigt werden, sind offiziell zugewiesen.
  • Sie dürfen nur eine OID pro Person registrieren . Mit Ihrer Root-OID können Sie sich unendlich viele untergeordnete OIDs zuweisen.
  • Anstatt Ihre neu generierte OID direkt für einen bestimmten Zweck zu verwenden, empfehlen wir Ihnen, für jeden Zweck eine Unter-OID zu delegieren. Nach dieser Strategie benötigen Sie nur eine Ihnen zugewiesene Wurzel-OID.
  • Bitte registrieren Sie eine OID nur, wenn Sie sicher sind, dass Sie einen OID-Namensraum benötigen.
  • Lesen Sie die folgenden Punkte sorgfältig durch, um sicherzustellen, dass Sie einen geeigneten OID-Namensraum für Ihre Anforderungen erhalten:
    • Wenn Sie ein Unternehmen vertreten, sollten Sie stattdessen eine OID unter 1.3.6.1.4.1 registrieren (weitere Informationen).
    • Wenn Sie nur eine OID für ein Beispiel in einem Dokument benötigen, verwenden Sie bitte die OID 2.999 (keine Registrierung erforderlich).
    • Wenn Sie sicher sind, dass Ihre Zielanwendung OIDs mit 128-Bit-Nummern zulässt (was für die meisten ASN.1-Implementierungen nicht gilt), können Sie eine global eindeutige OID (UUID OID) unter OID 2.25 registrieren (siehe Generieren einer solchen OID und weitere Informationen zu bekannten Einschränkungen ). Sie können jedoch weiterhin eine OID von diesem Dienst anfordern, da diese kürzer und leichter zu merken ist.
    • Wenn Sie das Active Directory-Schema erweitern möchten, sollten Sie stattdessen eine OID von Microsoft anfordern (weitere Informationen).
    • Wenn Sie ein Land vertreten, sollten Sie stattdessen eine Unter-OID von 1.2 (veraltet) oder 2.16 (empfohlen) anfordern (weitere Informationen).
    • Wenn Sie einen Telekommunikationsbetreiber vertreten, sollten Sie stattdessen eine Unter-OID von 0.2 anfordern (weitere Informationen).
    • Wenn Sie einen Netzbetreiber vertreten, sollten Sie stattdessen eine Unter-OID von 0.3 anfordern (weitere Informationen).
  • Eine registrierte OID kann niemals abgemeldet oder neu zugewiesen werden. Sie wird immer für die Öffentlichkeit sichtbar bleiben.
  • Sie erklären sich damit einverstanden, dass Ihre OID einschließlich Ihrer E-Mail-Adresse in diesem öffentlichen Register sowie auf oid-info.com und anderen öffentlichen Diensten veröffentlicht wird.
  • Die von Ihnen eingegebenen Daten können jederzeit mit dem Passwort, das Sie bei der Aktivierung eingeben werden, geändert werden.
  • Bitte verwenden Sie ein sicheres Passwort und geben Sie das Passwort nicht an andere Personen weiter.
  • Sie können keine bevorzugte Nummer für Ihre OID auswählen. Ihnen wird eine Nummer fortlaufend zugewiesen.
  • Diese automatisierte Registrierungsstelle weist keine alphanumerischen Kennungen oder IRI-Namen (Internationalized Resource Identifier) zu, da dieses System vollständig automatisiert ist und nur die E-Mail-Adresse überprüft wird.
  • Ihre neue OID wird folgender OID untergeordnet: {{ROOT_OID}} mit ASN.1-Notation { {{ROOT_OID_ASN1}} } und OID-IRI-Notation: {{ROOT_OID_IRI}}
  • Wenn Sie Fragen haben oder auf Fehler oder Probleme stoßen, wenden Sie sich bitte an {{ADMIN_EMAIL}} (auf Englisch oder Deutsch).
